MogiBot


About

MogiBot is written with the intent to help organize mogi. However, the bot is not just limited to mogi since it is configurable.

Invite

If you would like to invite MogiBot to your server, use the following:

Invite MogiBot

Roles

If you wish to grant administrator access, create and assign a role called administrator. If you wish to grant moderator access, create and assign a role called moderator.

Administrator (includes Moderator and public commands)
!setconfig - list of properties that can be configured on the bot
!setconfig <property> <value> - configure the bot's property with a specified value

Moderator (includes public commands)
!notify <message> - send a notification message the players in the mogi
!remove <#> - remove player # from the mogi list
!end - end a mogi before the required time

Public
!start - start a mogi
!end - end a mogi
!can or !c - join the mogi
!drop or !d - drop from the mogi
!list or !l - list who is in the mogi
!s - list who is in the mogi (temporary)
!mogilist - list of all the mogi in the same server

Configure

If you want to configure the bot for your server, you will need to have the administrator role. You can type !setconfig to get list of properties that can be configured on the bot.

Properties
room_size <#> - room size for a full room (min: 2, max: 100, default: 12)
max_subs <#> - number of subs allowed after the room is full (min: 0, max: 100, default: 3)
inactive_message_after <#> - number of minutes before someone is considered inactive after joining the mogi (min: 5, max: 300, default: 25)
remove_inactive_from_mogi_after <#> - number of minutes after someone is sent an inactive notice before removing them from the mogi (min: 1, max: 300, default: 5)
minimum_time_before_mogi_end_after_full <#> - number of minutes after the mogi is full before someone can end the mogi (min: 0, max: 300, default: 40)

Example 1: Configure so there are only 10 players in the room
!setconfig room_size 10

Example 2: Configure to not allow subs
!setconfig max_subs 0

Example 3: Configure to send an inactive notification at 10 minutes and remove them after 15 minutes
!setconfig inactive_message_after 10
!setconfig remove_inactive_from_mogi_after 5

Example 4: Configure so anyone can end if the mogi is full NOT RECOMMENDED
!setconfig minimum_time_before_mogi_end_after_full 0
